# 9. Tech Stack

Our technology stack is chosen for security, scalability, and user experience.

## Smart Contracts (On-Chain)

- **Language**: Solidity (v0.8.x)
- **Framework**: Foundry (for development, testing, and deployment)
- **Libraries**:
  - **OpenZeppelin**: For robust implementations of `ERC20`, `Ownable`, and standard security modules.
  - **ReentrancyGuard**: Protects `donate` and `withdraw` functions from re-entrancy attacks.
- **Core Concepts**:
  - `CampaignFactory.sol`: Deploys individual campaign contracts (Factory Pattern).
  - `Campaign.sol`: Holds funds, manages state, and handles swaps.
- **Network**: Base (Sepolia Testnet / Mainnet)
- **Integrations**:
  - **IDRX**: The peg for all value in the system.
  - **Uniswap/Aerodrome Router**: For on-chain token swaps.

## Backend (Off-Chain)

- **Runtime**: Node.js
- **Indexer & API**: Ponder (for robust blockchain indexing and GraphQL API).
- **Framework**: Next.js API Routes (Serverless).
- **Database**: PostgreSQL (managed via Ponder).
- **Payments**: Midtrans SDK (for handling Indonesian Fiat payments).

## Frontend (User Interface)

- **Framework**: Next.js (React)
- **Language**: TypeScript
- **Styling**: Tailwind CSS (for rapid, responsive UI design).
- **Web3 Connectivity**:
  - **OnchainKit**: For Base-native components.
  - **RainbowKit/Wagmi**: For wallet management and contract hooks.
  - **Viem**: For low-level blockchain interaction.
