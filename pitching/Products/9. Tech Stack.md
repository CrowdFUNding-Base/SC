# 9. Tech Stack

Our technology stack is chosen for security, scalability, and user experience, leveraging the latest Solidity features and the Base ecosystem.

## Smart Contracts (On-Chain)

- **Language**: Solidity `^0.8.30`
- **Framework**: Foundry (for development, testing, and deployment)
- **Architecture**: Singleton Pattern (Gas Efficient)
- **Contracts**:
  - **`Campaign.sol`**: The core protocol logic. It acts as a singleton manager for all campaigns to save gas (vs deploying a new contract for every campaign).
    - **Features**: Campaign creation, processing donations (ETH/ERC20), handling auto-swaps to **IDRX** via `MockSwap`, and secure withdrawals.
    - **Security**: Implements `ReentrancyGuard` and `SafeERC20` for robust asset handling.
  - **`Badge.sol`**: The Gamification Layer.
    - **Standard**: `ERC721` (NFT).
    - **Features**: Mints unique "Impact Badges" to donors as permanent, evolving proof of contribution.
- **Libraries**:
  - **OpenZeppelin**: `ERC721`, `Ownable`, `ReentrancyGuard`, `IERC20`, `SafeERC20`.
- **Integrations**:
  - **IDRX**: The native settlement token for all campaigns.
  - **DEX Router**: For atomic swaps of global assets (ETH/USDC) into IDRX.

## Backend (Off-Chain)

- **Runtime**: Node.js
- **Indexer & API**: Ponder (for robust blockchain indexing and GraphQL API).
- **Framework**: Next.js API Routes (Serverless).
- **Database**: PostgreSQL (managed via Ponder).
- **Payments**: Midtrans SDK (for handling Indonesian Fiat payments/QRIS).

## Frontend (User Interface)

- **Framework**: Next.js (React)
- **Language**: TypeScript
- **Styling**: Tailwind CSS (for rapid, responsive UI design).
- **Web3 Connectivity**:
  - **Base OnchainKit**: For seamless "Seedless" onboarding and profile management.
  - **RainbowKit/Wagmi**: For wallet management and contract hooks.
  - **Viem**: For low-level blockchain interaction.
